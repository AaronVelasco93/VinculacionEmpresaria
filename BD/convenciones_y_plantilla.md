# Convenciones de nombres y commits

Para mantener orden, claridad y trazabilidad en el desarrollo del
proyecto, todos los equipos deberán seguir las siguientes convenciones.

## 1. Convenciones de nombres de archivos y carpetas

-   Usar nombres descriptivos.
-   Evitar espacios en blanco.
-   Usar guion medio `-` o guion bajo `_`.
-   No usar caracteres especiales ni acentos.

Ejemplos correctos: - documento-requerimientos.md - casos-de-uso.pdf -
diagrama-base-datos.png

## 2. Convención por tipo de entrega

Documentación: - requerimientos-equipo-1.pdf - casos-uso-equipo-2.pdf

Código: - frontend/ - backend/ - database/

Evidencias: - evidencias-semana-01/

## 3. Convención de ramas

-   main
-   develop
-   feature/nombre-funcionalidad
-   fix/nombre-correccion

## 4. Convenciones de commits

Formato: tipo: descripcion breve

Tipos: - feat - fix - docs - style - refactor - test - chore

Ejemplos: - feat: agregar formulario de reportes - fix: corregir
validacion

## 5. Buenas prácticas

-   Commits pequeños y frecuentes
-   Mensajes claros
-   No mezclar cambios

# Plantilla base para entregas

## Estructura general

nombre-del-proyecto/ - README.md - docs/ - src/ - database/ -
evidencias/ - entregas/

## README.md

Debe incluir: - Equipo - Integrantes - Objetivo - Descripción - Estado

## docs/

-   Documento de requerimientos
-   Casos de uso
-   Diagramas

## src/

Código fuente del proyecto

## database/

-   esquema.sql
-   datos-prueba.sql

## evidencias/

-   capturas
-   pruebas

## entregas/

-   semana-01
-   semana-02
-   entrega-final

## Criterios mínimos

-   Orden
-   Claridad
-   Evidencias
-   Coherencia
