## Tecnologías

### Node.js (Servidor)
Entorno asíncrono y no bloqueante, ideal para múltiples peticiones concurrentes. Unifica el stack al usar JavaScript/TypeScript en todo el proyecto.

### Next.js (Backend)
Proporciona API Routes para crear endpoints sin servidor adicional, renderizado híbrido (SSR/SSG) para optimizar SEO y rendimiento, y middleware para autenticación.

### React (Frontend)
Biblioteca para construir interfaces de usuario basadas en componentes reutilizables. Virtual DOM que optimiza las actualizaciones, y hooks para manejar estado y efectos secundarios.

### HTML5 + CSS3 + Tailwind (Frontend)
HTML semántico y accesible, CSS3 con Flexbox/Grid para diseño responsivo, y Tailwind CSS para desarrollo rápido con utilidades y diseño consistente.

### Prisma (ORM)
Type-safe, genera tipos automáticos desde la base de datos, migraciones declarativas y sintaxis intuitiva que reduce código repetitivo.

### TypeScript / JavaScript
TypeScript aporta tipado estático que previene errores en tiempo de desarrollo y mejora el mantenimiento. JavaScript como base ejecutable en cualquier navegador.

### Docker (Contenedores)
Empaqueta la aplicación y sus dependencias en contenedores aislados, garantizando el mismo entorno en desarrollo, pruebas y producción.

## Justificación General

**Node.js** destaca por su eficiencia en operaciones de entrada/salida y su enorme ecosistema de paquetes, lo que acelera el desarrollo del servidor. **Next.js** simplifica el flujo de trabajo al combinar frontend y backend en un solo proyecto, ofreciendo rutas de API integradas y múltiples estrategias de renderizado que mejoran la experiencia de usuario. Sobre Next.js se construye **React**, que permite crear componentes reutilizables y gestionar el estado de la interfaz de manera eficiente gracias a su Virtual DOM, mientras que **HTML5** y **CSS3** garantizan estructura semántica y diseños adaptables, y **Tailwind** acelera la maquetación con clases reutilizables sin necesidad de escribir CSS personalizado. **Prisma** facilita la interacción con la base de datos mediante consultas intuitivas y migraciones automáticas, reduciendo el código repetitivo. **TypeScript** aporta robustez al conjunto detectando errores antes de ejecutar el código, y **JavaScript** actúa como el lenguaje base que cohesiona todo el ecosistema. Finalmente, **Docker** encapsula toda la aplicación en contenedores portables, eliminando el "funciona en mi máquina" y facilitando el despliegue consistente en cualquier entorno.
